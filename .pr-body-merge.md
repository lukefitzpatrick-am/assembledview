- **Shared portfolio service** — Extracted portfolio query and plan fetch into `lib/snowflake/portfolio-pacing-service.ts` and `lib/pacing/plan/fetchPortfolioPlan.ts`; portfolio route is thinner and delegates to shared helpers.
- **Pacing maths module** — New `lib/pacing/maths/` with date arithmetic, DIV0-safe helpers, spend/pacing derivations, status classification, and `computePacing` entry point (aligned with SQL `V_LINE_ITEM_PACING` semantics).
- **Line-item composition** — `buildLineItemPacingRows` merges plan + delivery data; `applyOverviewFilters` and `resolveClientSlugs` handle overview filtering and client ID → slug resolution.
- **API cutover** — `/api/pacing/line-items` repointed at consolidated path; portfolio union extended with **search** line items (`media_plan_search`).
- **UI simplification** — `app/pacing/portfolio/page.tsx` and related client code lean on shared data context instead of inline logic.
- **Tests & smoke** — Unit tests under `lib/pacing/maths/__tests__/` and `lib/pacing/composition/__tests__/`; new `npm run test:pacing-maths`; PowerShell smoke script `scripts/smoke/pacing-portfolio.ps1` for `POST /api/pacing/portfolio`.

### 2. Behavioural Campaign Planner mock (`feature/behavioural-planner-mock`)

- **New route:** `/tools/behavioural-planner` — four-stage UI mock (scaffold → form → results → polish).
- **BCS engine** — `lib/bcs-engine.ts` scores brief/audience inputs against seeded channel mix and cultural moments.
- **Components** — Brief, objective, audience, weights, metric cards, channel mix table, Ava narration, cultural moments, responsive tabs, budget input, reset.
- **Data:** Seeded mock data only; no backend persistence.
